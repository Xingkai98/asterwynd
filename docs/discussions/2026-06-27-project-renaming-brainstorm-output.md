# 命名候选

## 总体命名策略

这个项目的核心叙事是：**在复杂代码系统中辨认方向、找到改动路径、反复验证、留下证据**。名字不需要直白绑定 coding agent，但需要能承载这个叙事。

本次头脑风暴围绕 9 个语义方向发散：

1. **罗盘与导航** — 方向、指引、到达
2. **验证与证据** — 真值、trace、证明
3. **线索与连接** — 在迷宫中找到路径
4. **工艺与建造** — 工匠、锻造、铆接
5. **光与清晰** — 照亮、显影、揭示
6. **抽象代号** — 短、好听、像独立产品名
7. **中国元素轻融入** — 轻量拼音元素，不僵硬
8. **自然意象** — 风、星、石、水
9. **代码空间隐喻** — 语法、记号、版本

相比功能描述型名字（PatchRivet、Diffwright），代号型名字（Orivane、Talyra）更耐扩展，也更像可以长期维护的开源项目。以下候选以代号型为主，功能型作为对照保留。

### Top 10 细评

#### 1. Orivane — 软件变更的风向标

读音：OH-ree-vayn，清晰、两音节半、无难发音辅音簇。

故事：Ori 有 origin（起点）、orientation（方向）两层联想；vane 是风向标，暗示在当前代码状态下判断"往哪走"。Coding agent 的核心任务就是：理解仓库现状 → 判断改动方向 → 执行改动 → 验证 → 留下证据。Orivane 可以作为整个系统的名字——不只是 agent 本身，更是包含 runtime、工具系统、trace、benchmark 的完整工作台。

Tagline：
- EN: Navigate change. Prove the fix.
- CN: 找到起点，判断方向，验证修复。

为什么排第一：没有硬伤。读音、含义、故事、扩展性、搜索唯一性都在高水平。唯一风险是 `orivane` 用户已占用，但这对仓库名使用无影响。

#### 2. Elarive — 抵达可验证的结果

读音：EH-lah-reev，像 "ever" + "arrive"，非常顺口。

故事：Elarive 同时唤起 arrive（抵达）、alive（鲜活）、drive（驱动）三个正向词。Coding agent 从接收模糊任务（issue/PRD）到产生可验证结果（test-passing patch）的过程，就是一个"elarive"的过程——驱动自己抵达可验证终点。

Tagline：
- EN: Drive through complexity. Arrive at proof.
- CN: 穿过复杂，抵达确定。

为什么排第二：朗朗上口不输 Orivane，搜索潜力更干净（0 命中）。故事不如 Orivane 有层次感，但"抵达/驱动"的单一隐喻足够清晰。

#### 3. Talyra — Clean Slate

读音：TAH-lee-rah，干净、女性化音色、像一个成熟产品。

故事：Talyra 不携带任何既有语义，这本身是一个叙事策略——项目名不需要解释自己。它就是一个"名字"，意义由项目本身填充。类比：Pi、Kubernetes、Elasticsearch 都是名字先于含义。

Tagline：
- EN: A local agent system for software work.
- CN: 本地软件工程 agent 系统。

为什么排第三：搜索潜力最好（0 有效命中），最像"长期品牌名"。但含义轻意味着前期需要更多品牌建设。

#### 4. Veyline — 路径之线

读音：VAY-lyne，两个音节，干脆利落。

故事：Vey（来自 via/way，道路/路径）+ line（线段/基准线）。在日常开发里，coding agent 就是在代码空间中画出一条线——从问题到修复的那条路径。Veyline 暗示这条线是可追踪、可验证的。

Tagline：
- EN: Trace the line. Prove the change.
- CN: 循线溯源，证明变更。

为什么排第四：简洁有力，比 Orivane 更偏工具感但还在可接受范围内。0 命中，搜索非常干净。

#### 5. Orivora — 有生命的系统

读音：OH-ree-VOH-rah，轻重音交替，有节奏感。

故事：Orivora 是 Orivane 的"更大胆的弟弟"。vora 后缀让人想到 -vore（食性），比如 omnivore、carnivore，暗示这个系统"吞进去的是任务，吐出来的是可验证的变更"。比 Orivane 更像一个 AI-native 时代的名字。

Tagline：
- EN: Consume tasks. Produce proof.
- CN: 入则任务，出则证明。

为什么排第五：想象空间大，但 -vora 后缀的"吞噬"联想不是每个人都喜欢。0 命中。

#### 6. Verivane — 真值风向标

读音：VEH-ree-vayn，与 Orivane 形成清晰的家族对照。

故事：Veri 来自 verity（真值）、verify（验证）、veritas（拉丁语真理）。如果说 Orivane 强调"方向"，Verivane 强调"方向的正确性"。对偏测试/benchmark/verification 方向的 coding agent 来说，这是更精准的隐喻。

Tagline：
- EN: Pointing to the truth of change.
- CN: 指向变更的真相。

为什么排第六：含义正向极强，但比 Orivane 更"技术化"，想象力略弱。0 命中。

#### 7. Tracery — 追踪的技艺

读音：TRAY-suh-ree，像 "tracer" + "y"，有工艺感。

故事：Tracery 把 trace 变成一个"技艺"（类比 archery 射箭术、masonry 石匠术）。项目主线——记录 trace、diff、工具调用和 benchmark 证据——就是一项"追踪的技艺"。这个词真实存在（指哥特建筑的花窗格），借用到 coding agent 语境中非常有辨识度。

Tagline：
- EN: The craft of tracing change.
- CN: 追踪变更的技艺。

为什么排第七：故事潜力可能是 Top 10 里最强的。风险：tracery 的原有含义（花窗格）可能导致含义混淆；三音节。

#### 8. Lucidora — 清晰的洞见

读音：loo-sih-DOH-rah，轻重音在第三音节，读起来像一段旋律。

故事：Lucid（清晰、透彻）+ ora（光芒/氛围）。Coding agent 在复杂代码空间里做的事情，本质上就是让不透明的变更过程变得清晰透明。Lucidora 暗示"在迷雾中照进一束清晰的光"。

Tagline:
- EN: Clarity through complexity.
- CN: 穿过复杂的清晰。

为什么排第八：正向含义极其强大。扣分在三音节和略正式的音色——不像 `pi` 或 `hermes` 那么轻盈。

#### 9. Cluestone — 关键线索

读音：KLOO-stohn，两个重音节，干脆。

故事：Clue（线索）+ Keystone（拱心石/基础石）。拱心石是建筑中最后放入、但最关键的一块石头——放入它，整个结构才能站立。Coding agent 就是在代码仓库中找到那块 keystone：关键的改动、正确的 patch，让整个修复成立。

Tagline:
- EN: The key clue in complex code.
- CN: 复杂代码中的关键线索。

为什么排第九：故事性和画面感很强，适合面试叙事。风险：偏推理/侦探游戏感，可能被误认为游戏名。0 命中。

#### 10. Asterwynd — 星与风

读音：ASS-ter-wynd，两音节。

故事：Aster（星辰）+ Wynd（风，古拼法）。星辰代表方向、导航、恒久；风代表变化、流动、穿越。在代码仓库的星空中，Asterwynd 是那个能辨认星座（代码结构）、穿越星风（变更不确定性）的导航者。

Tagline:
- EN: A software navigator through change.
- CN: 穿越变更的软件导航者。

为什么排第十：画面感是全列表最强的一个。适合做 logo 和视觉叙事（星图 + 风的意象）。唯一扣分：wynd 的 fantasy 拼法可能导致拼写错误（被写成 wind）。

## 最终建议

### 如果追求稳妥和平衡 → `Orivane`

Orivane 在各项指标的雷达图上最接近正五边形。它在当前项目讨论中已经过五轮推敲，故事、tagline、风险都梳理得很清楚。如果不需要更多发散，Orivane 可以直接进入下一阶段（PyPI、域名、文档 rename 规划）。

### 如果追求最大胆的代号感 → `Talyra` 或 `Elarive`

Talyra 最像一个"成熟产品该有的名字"——短、干净、没有任何技术包袱。Elarive 则在顺口程度上不输 Orivane，同时更空灵。两者都需要品牌建设，但起点更高。

### 如果追求与 trace/benchmark 主线最强的共振 → `Tracery`

Tracery 的双关含义（trace 追踪 + -ery 技艺）与项目的差异化定位天然贴合。如果你在面试时最想讲的是"trace、diff、benchmark 闭环"，Tracery 会是一个极佳的叙事载体。

### 如果追求中国元素的自然融入 → `Xingvane` 或 `Minglark`

如果团队认为中国元素是品牌差异化的一部分，Xingvane（星 + 风向标）和 Minglark（明 + 晨光）是两个不显生硬的选择。需要注意英语母语者对 `xing` 和 `ming` 的发音。

### 建议下一步

1. 选出 2-3 个最终候选，不要在 10 个里徘徊。
2. 对每个最终候选逐一执行完整的 PyPI / npm / Docker Hub / 域名 / 搜索引擎复查。
3. 让不止一个人大声读 10 遍，感受发音疲劳度。
4. 给每个候选写一段 3 分钟的面试开场白，看哪个讲起来最自然。
5. 决定后，先做品牌文本 rename（README、文档），不要急着改 import path。
