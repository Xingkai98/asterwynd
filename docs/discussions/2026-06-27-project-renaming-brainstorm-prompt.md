# 项目重命名头脑风暴 Prompt

本文档用于交给另一个模型，请它为当前项目继续发散命名方案。

## 背景

当前项目名是 `MyAgent`，问题是过于普通：

- GitHub 和搜索引擎里难以检索。
- 很多项目都叫 MyAgent，缺少辨识度。
- 面试或对外介绍时没有记忆点。

项目正在考虑更名。请不要只做字面替换，而是先理解项目愿景，再提出名字。

## 项目愿景

这个项目不是通用聊天机器人，也不是单一 patch 工具。它希望成为一个本地软件工程 Agent 系统：

- 理解代码仓库结构和上下文。
- 通过工具读取、搜索、分析和修改代码。
- 运行测试或验证命令。
- 记录 trace、diff、工具调用和 benchmark 证据。
- 把模糊的软件任务推进到可验证的软件变更。

可以把它理解为：

> 一个在复杂代码系统里辨认方向、找到改动路径、反复验证并留下证据的本地 Agent。

更短的定位：

> It turns repository tasks into traceable, test-backed software changes.

## 命名目标

请优先满足以下标准：

1. **朗朗上口**：读起来顺，最好 2-4 个音节。
2. **正向含义**：联想到方向、抵达、线索、验证、修复、清晰、可靠、探索、创造等正向意象。
3. **新颖但不过分怪**：像一个可以长期维护的开源项目名，而不是临时代号。
4. **不必直白绑定 coding agent**：名字可以更抽象、更有想象力，但需要能讲出和项目愿景相关的故事。
5. **国际化可读**：英文读者能读，中文使用者也不觉得别扭。
6. **搜索友好**：尽量避免明显常见词、热门神话原名、已有大项目名。

## 可接受的命名风格

可以从这些方向发散：

- 软件变更的风向标、罗盘、导航仪。
- 代码迷宫里的线索、路径、星图。
- 不知疲倦的工程伙伴，但避免“徒劳重复”的负面意象。
- 可验证改动的证明链。
- 本地 Agent runtime / software workbench 的抽象代号。
- 更像 `Hermes`、`OpenClaw`、`Pi` 那种有故事或意象的名字。
- 可以轻微融入中国元素，但不要生硬拼音化，也不要太像中文互联网产品名。

## 已讨论过的候选

### 当前最强候选：`Asterwynd`

故事：

- `Aster` 来自星辰意象。
- `Wynd` 是 wind 的古英语/诗歌拼法。
- 整体可解释为“以星辰定向、乘风穿越复杂代码空间的导航者”。

英文 slogan：

> Navigate by stars. Prove with traces.

中文 slogan：

> 以星为引，变更有证。

GitHub 复查：

- 仓库名搜索 `Asterwynd` / `asterwynd`：0 命中。
- `github.com/asterwynd` 用户/组织名未被占用。
- 常规拼法 `Asterwind` 的 GitHub 用户名已被空账号占用，但 `Asterwynd` 完全干净。

### 强备选：`Orivane`

故事：

- `Ori` 可联想到 `origin` 和 `orientation`。
- `vane` 是风向标。
- 整体可解释为“软件变更的风向标”：找到起点，判断方向，验证修复。

英文 slogan：

> Navigate change. Prove the fix.

中文 slogan：

> 找到起点，判断方向，验证修复。

GitHub 复查：

- 仓库名搜索 `Orivane` / `orivane`：0 命中。
- `github.com/orivane` 用户名已被占用，但该用户公开仓库、gist、关注、粉丝和公开事件均为空。

## 避免的方向

请尽量避免：

- 直接使用 `MyAgent`、`CodeAgent`、`RepoAgent` 这类泛名。
- 过于功能化的名字，如 `PatchFixer`、`CodeRepairBot`。
- 重名严重的神话原名，如 `Ariadne`、`Daedalus`、`Hermes`、`Pythia`、`Mimir`。
- 含义偏负面的名字，如直接用 `Sisyphus`。它有坚持意味，但也有徒劳重复的含义。
- 拼写太难、读音不稳定、过度 fantasy 的名字。

## 输出要求

请给出 30-50 个候选名，按类别组织。

每个候选请包含：

- 名字。
- 读音建议。
- 一句话含义。
- 和项目愿景的连接。
- 英文 tagline。
- 中文 tagline。
- 风险或缺点。
- 初步 GitHub 搜索风险判断：高 / 中 / 低。如果无法联网，请基于常见度推测，并明确说明这是推测。

请最后选出 Top 10，并说明排序理由。

## 评分标准

请按 1-5 分给每个 Top 10 候选评分：

| 维度 | 说明 |
| --- | --- |
| 朗朗上口 | 是否好读、好记、适合口头介绍。 |
| 含义正向 | 是否有积极、可靠、探索、验证、创造等联想。 |
| 故事性 | 是否能讲出一个和项目愿景相关的故事。 |
| 可扩展性 | 是否不会把项目限制成某个单一工具。 |
| 搜索潜力 | 是否看起来不容易和已有项目重名。 |
