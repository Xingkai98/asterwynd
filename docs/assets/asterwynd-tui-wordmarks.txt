## TUI Recommendation

Use Option B as the default TUI startup masthead. It keeps `w`, `y`, `n`, and
`d` at the same visual height and makes the final `ynd` unambiguous.

Keep slanted wordmarks as exploration only until the final `n` remains readable
in real terminal rendering.

Use Option D when terminal width is below 92 columns.

Use Option F when stdout is non-interactive, logs are captured, or terminal
height is constrained.

If any stylized masthead makes the final `ynd` ambiguous, render a plain
`Asterwynd` line directly underneath it. Readability beats ornamentation.

Suggested colors:

- Wordmark: bright cyan to pale violet, or plain bold if gradients are not available.
- Star/accent glyphs, if added around the masthead: warm amber.
- Slogan: dim slate / gray.

Avoid embedding ANSI escape sequences in this source file. Keep it plain text
and let the TUI renderer apply styles.
